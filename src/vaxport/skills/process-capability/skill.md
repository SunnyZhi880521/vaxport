# 过程能力评估 SKILL

## 触发条件

当用户询问以下内容时，应使用本 SKILL：

- "过程能力"、"Cpk"、"process capability"、"能力指数"
- "这个参数稳定吗"、"能力够不够"、"达标吗"
- 涉及批次数据的变异性评估、规格限对比
- "联合合格率"、"多指标过程能力"

## 领域知识（Agent 必须知道）

### 核心概念

- **Cp**（过程精密度）：衡量过程变异与规格限的关系，不考虑偏移
  - Cp = (USL - LSL) / (6σ)
- **Cpk**（过程能力指数）：同时考虑过程中心偏移
  - Cpk = min((USL - μ)/(3σ), (μ - LSL)/(3σ))
- 至少需要 **20 批**数据才有统计意义
- 数据必须处于**统计受控状态**，否则 Cpk 无意义

### 判定标准（行业通用，可根据企业标准调整）

| Cpk 范围 | 判定 | 建议行动 |
|---------|------|---------|
| ≥ 1.67 | 优秀 | 维持现状，可考虑放宽检验 |
| 1.33 ~ 1.67 | 充分 | 维持现状，常规监控 |
| 1.0 ~ 1.33 | 基本合格 | 增加监控频率，关注趋势 |
| < 1.0 | 不足 | 启动 CAPA，分析根本原因 |

### 前置条件

- 数据必须处于**统计受控状态**，否则 Cpk 无意义
- 需要先检查控制图，确认无特殊原因变异
- 规格限必须从可靠来源获取

## 分析框架（按此顺序执行）

### 第一步：数据准备

**目标**：收集目标参数的历史批次数据

**要点**：
- 需要时序数据（按批次日期排序）
- 记录数据量，< 20 批时在结论中明确标注
- 如果存在明显的工艺变更节点（如设备更换、原料供应商变更），考虑分段分析
- 识别所有相关产品和参数

**数据获取方向**：查询目标参数的历史批次数据

### 第二步：统计受控检查

**目标**：确认数据是否处于统计受控状态

**要点**：
- 使用 Western Electric 规则检测控制图异常
- 如果触发规则，说明存在特殊原因变异
- 不受控时，Cpk 计算结果不可靠，应先调查异常原因

**分析方向**：检查数据是否处于统计受控状态

### 第三步：过程能力计算

**目标**：计算 Cp、Cpk

**关键约束**：
- ⚠️ **规格限必须从可靠来源获取**，优先级：
  1. 用户明确指定
  2. 数据库中的产品标准表/注册文件
  3. 询问用户
- ❌ **禁止自行假设规格限**

**分析方向**：计算过程能力指数

### 第四步：趋势分析

**目标**：检查是否存在显著的时间趋势

**要点**：
- 使用回归分析检查趋势
- 如果存在显著趋势，当前 Cpk 不代表未来能力
- 趋势明显时，建议预测何时可能超标

**分析方向**：检查过程能力的时间趋势

### 第五步：多参数联合分析（如适用）

**目标**：评估多个质量指标的联合合格率

**要点**：
- 计算各指标单独通过率
- 计算联合概率（所有指标同时合格的概率）
- 识别"瓶颈"指标（通过率最低、拖累整体的指标）
- 敏感性分析：提升联合合格率应优先改善哪个指标

## 输出规范

### 必须包含的章节

1. **数据概览**：批次数量、时间范围、基本统计（均值、标准差、CV%）
2. **受控状态**：控制图 + 异常点说明
3. **过程能力**：Cp、Cpk 值 + 判定结论
4. **趋势分析**：如有显著趋势需说明
5. **结论与建议**：综合评价 + 下一步行动

### 必须生成的图表（最少 3 张，最多 5 张）

调用 generate_chart 时必须传入 **section** 参数，值为当前分析步骤名称：
- 第一步"数据准备" → section="数据准备"
- 第二步"统计受控检查" → section="统计受控检查"
- 第三步"过程能力计算" → section="过程能力计算"
- 第四步"趋势分析" → section="趋势分析"
- 第五步"多参数联合分析" → section="多参数联合分析"

每个 section 最多 5 张图表。超过后该章节不再生成新图表，直接撰写文字结论。

- ✅ 控制图（I-MR 或 Xbar-R，标注异常点）
- 📊 趋势图（如存在显著趋势）
- 📊 直方图 + 规格限（可选，直观展示分布）

⚠️ **收敛约束**：图表上限 5 张。超过后不再生成新图表，直接撰写结论。

## 行为约束（硬性规则）

### 必须做的

- ✅ 规格限不确定时，询问用户或从数据库查询
- ✅ 数据量 < 20 批时，结论加限定语"样本量有限，仅供参考"
- ✅ 不受控时，先说明异常再决定是否计算 Cpk
- ✅ 多产品时，逐个产品分析并对比

### 禁止做的

- ❌ 不要猜测规格限（不能假设"效价规格通常是 6-8"）
- ❌ 不要在数据明显不受控时直接给出 Cpk 结论
- ❌ 不要用非时序数据计算 Cpk（批次顺序不能乱）
- ❌ 不要给出过于绝对的结论（"过程能力完全没问题"）

## 常见错误防范

| 错误场景 | 错误做法 | 正确做法 |
|---------|---------|---------|
| 用户未提供规格限 | 假设"通常效价规格是 6-8" | 询问用户或查产品标准表 |
| 数据有 3 个异常点 | 直接计算 Cpk | 先说明异常点，调查原因 |
| 只有 15 批数据 | "过程能力充分" | "基于现有 15 批数据，Cpk=1.2，但样本量有限..." |
| 存在明显下降趋势 | 只报告当前 Cpk | 报告 Cpk + 趋势预警 + 预测何时可能不达标 |
| 多产品混在一起算 | 一个总 Cpk | 每个产品单独计算 Cpk |
| 忽略单位差异 | 不同单位直接比较 | 注意每个参数的单位和规格限 |
