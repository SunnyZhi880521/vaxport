# 偏差分析与 CAPA 评估 SKILL

## 触发条件

当用户询问以下内容时，应使用本 SKILL：

- "偏差"、"异常"、"deviation"、"OOS"
- "帕累托分析"、"Top N 问题"、"主要偏差类型"
- "CAPA 有效性"、"CAPA 闭环"、"纠正措施有没有效"
- "偏差复发"、"同类问题是否再次出现"
- "偏差分类"、"按严重程度"

## 领域知识（Agent 必须知道）

### 偏差分级标准（GMP 通用）

| 等级 | 定义 | 典型处理 |
|------|------|---------|
| Critical（严重） | 可能影响产品质量或患者安全 | 立即调查，QA 审批，必须 CAPA |
| Major（主要） | 可能影响产品质量但风险可控 | 48h 内启动调查，CAPA |
| Minor（次要） | 不影响产品质量 | 记录，定期回顾 |

### CAPA 生命周期

```
偏差发现 → 分级 → 根因分析 → CAPA 制定 → CAPA 执行 → 有效性验证 → 关闭
```

### 帕累托原则

- 80/20 法则：80% 的偏差通常由 20% 的原因类型造成
- Top 3 偏差类型应占累计 60-80%
- 帕累托图：按频率降序排列 + 累计百分比线

### ICH Q10 对 CAPA 的要求

- CAPA 系统应覆盖纠正措施和预防措施
- 需验证 CAPA 有效性（关闭后同类偏差不再发生）
- 定期回顾 CAPA 整体健康度

## 分析框架（按此顺序执行）

### 第一步：偏差总览

**目标**：获取偏差全貌

**要点**：
- 按类型/严重度/产品/时间维度汇总
- 总偏差数、各级别数量及占比
- 时间分布（按月/季度统计，看趋势）

**数据获取方向**：查询偏差相关数据

### 第二步：帕累托分析

**目标**：识别关键少数问题

**要点**：
- 按偏差类型排序，计算累计百分比
- 识别 Top 3 类型（占累计 60-80%）
- 生成帕累托图

**可视化方向**：用图表展示偏差类型分布

### 第三步：CAPA 关联分析（如适用）

**目标**：评估 CAPA 体系有效性

**要点**：
- CAPA 总览：状态分布（进行中/已完成/逾期）
- 按时完成率
- 复发检查：CAPA 关闭后，同类偏差是否再现
- 无效 CAPA 识别：关闭但仍复发的案例
- 根因分析：根因没找对 vs 措施没落实 vs 新原因

### 第四步：趋势与关联

**目标**：深度分析

**要点**：
- 偏差率趋势（升/降/稳定）
- 与产品/设备/人员/季节的关联
- 高风险组合识别

### 第五步：结论与建议

**目标**：给出可执行建议

**要点**：
- 主要问题总结
- CAPA 体系健康度评价
- 优先改进领域
- 引用 ICH Q10 或 GMP 条款

## 输出规范

### 必须包含的章节

1. **偏差总览**：总数、分级统计、时间分布
2. **帕累托分析**：Top N 类型、累计占比
3. **CAPA 分析**（如适用）：完成率、有效性、复发情况
4. **趋势分析**：偏差率变化趋势
5. **结论与建议**：主要问题 + 改进建议

### 必须生成的图表（最少 3 张，最多 5 张）

调用 generate_chart 时必须传入 **section** 参数，值为当前分析步骤名称：
- 第一步"偏差总览" → section="偏差总览"
- 第二步"帕累托分析" → section="帕累托分析"
- 第三步"CAPA关联分析" → section="CAPA关联分析"
- 第四步"趋势与关联" → section="趋势与关联"
- 第五步"结论与建议" → section="结论与建议"

每个 section 最多 5 张图表。超过后该章节不再生成新图表，直接撰写文字结论。

- ✅ 帕累托图（偏差类型排序 + 累计百分比）
- 📊 偏差时间趋势图（按月统计）
- 📊 CAPA 状态饼图（如适用）

⚠️ **收敛约束**：图表上限 5 张。超过后不再生成新图表，直接撰写结论。
每个分析步骤最多产出 1-2 张图表。连续 2 轮只生成图表无新数据发现 → 停止分析，开始写结论。

## 行为约束

### 必须做的

- ✅ 偏差分级标准需从数据推断或引用 GMP 通用标准
- ✅ 帕累托分析必须包含累计百分比
- ✅ CAPA 有效性评估需要足够观察期（至少 6 个月）

### 禁止做的

- ❌ 不要在没有数据支持时断言"CAPA 有效"
- ❌ 不要把不同级别的偏差混在一起分析
- ❌ 不要忽略逾期 CAPA
