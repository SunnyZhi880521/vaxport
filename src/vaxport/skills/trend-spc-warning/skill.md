## 领域知识（Agent 必须知道）

### 控制图类型选择

| 数据类型 | 控制图 | 适用场景 |
|---------|--------|---------|
| 连续值（单值） | I-MR 图 | 批次检测值（效价、pH 等） |
| 连续值（子组） | Xbar-R 图 | 同一批次多个样品 |
| 计数值（不合格数） | p 图 / np 图 | 合格率/不合格率 |
| 计数值（缺陷数） | c 图 / u 图 | 缺陷数/缺陷率 |

### 控制限计算

```
I-MR 图：
  CL = X̄ (所有单值的均值)
  UCL = X̄ + 2.66 × MR̄
  LCL = X̄ - 2.66 × MR̄

  MR = |Xi - Xi-1| (移动极差)
  MR̄ = MR 的均值

Xbar-R 图：
  CL = X̿ (所有子组均的均值)
  UCL = X̿ + A2 × R̄
  LCL = X̿ - A2 × R̄

  A2 系数（n=5 时 A2=0.577，n=3 时 A2=1.023）
```

**关键约束**：
- 控制限是从数据计算的，**不是规格限**
- 规格限（USL/LSL）是质量标准，控制限是统计边界
- 两者不能混淆

### Western Electric 规则（完整 8 条）

将控制限到中心线的距离分为 3 个区域：
- A 区：2σ ~ 3σ
- B 区：1σ ~ 2σ
- C 区：0 ~ 1σ

| 规则 | 描述 | 含义 | 严重度 |
|------|------|------|--------|
| WE-1 | 1 点超出 3σ | 特殊原因变异 | Action |
| WE-2 | 连续 9 点在中心线同侧 | 过程均值偏移 | Action |
| WE-3 | 连续 6 点递增或递减 | 趋势 | Alert |
| WE-4 | 连续 14 点交替上下 | 系统性变异（两个过程混合） | Alert |
| WE-5 | 3 点中 2 点超过 2σ（同侧） | 过程变异增大 | Action |
| WE-6 | 5 点中 4 点超过 1σ（同侧） | 过程均值小偏移 | Alert |
| WE-7 | 连续 15 点在 ±1σ 内 | 数据分层（子组内变异 < 子组间） | Alert |
| WE-8 | 连续 8 点在 ±1σ 外 | 两个过程混合 | Alert |

### Nelson 补充规则

| 规则 | 描述 | 与 WE 关系 |
|------|------|-----------|
| Nelson-1 | 1 点超出 3σ | = WE-1 |
| Nelson-2 | 连续 9 点同侧 | = WE-2 |
| Nelson-3 | 连续 6 点递增/递减 | = WE-3 |
| Nelson-4 | 连续 14 点交替 | = WE-4 |
| Nelson-5 | 2/3 点超过 2σ | ≈ WE-5（更敏感） |
| Nelson-6 | 4/5 点超过 1σ | ≈ WE-6（更敏感） |
| Nelson-7 | 连续 15 点在 ±1σ | = WE-7 |
| Nelson-8 | 连续 8 点在 ±1σ 外 | = WE-8 |

### 趋势 vs 漂移

```
趋势（Trend）:
  ──↗───────  参数随时间单调变化
  原因：试剂老化、设备磨损、毒种传代

漂移（Shift）:
  ──────┬────  参数均值突然变化
        │
  原因：设备更换、原料批次变更、操作人员变更
```

**区分方法**：
- 趋势：回归斜率显著（p < 0.05），但前后两段均值差异不显著
- 漂移：前后两段均值差异显著（t 检验 p < 0.05），但段内无趋势

### 过程能力与 SPC 的关系

```
SPC 受控 → 可以计算 Cpk → 评估过程能力
SPC 不受控 → Cpk 无意义 → 先消除特殊原因
```

## 分析框架（按此顺序执行）

### 第一步：参数选择与数据准备

**目标**：确定分析对象

**要点**：
- 识别关键质量参数（CQA）：效价、pH、内毒素、纯度等
- 按产品分别收集数据
- 按时间排序（批次顺序）
- 数据量 < 20 点时标注"数据不足"

**数据获取方向**：查询 final_product_qc 或 in_process_tests 表

### 第二步：控制图构建

**目标**：建立控制图

**要点**：
- 计算中心线（CL）和控制限（UCL/LCL）
- 绘制 I-MR 图（单值）或 Xbar-R 图（子组）
- 标注规格限（USL/LSL）作为参考线

**关键约束**：
- ⚠️ 控制限从数据计算，不是规格限
- 如果数据中有已知的异常批次，考虑排除后重新计算控制限

### 第三步：规则检测

**目标**：应用 WE/Nelson 规则

**要点**：
- 逐条应用 8 条规则
- 记录每条规则的触发情况
- 区分 alert 和 action 级别
- 标记具体的触发位置（哪些批次）

### 第四步：趋势与漂移分析

**目标**：深度分析

**要点**：
- 线性回归检测趋势（斜率显著性）
- 分段分析检测漂移（前后均值比较）
- 如果有趋势，预测何时可能超出控制限/规格限
- 如果有漂移，定位变化时间点

### 第五步：多参数综合评估

**目标**：综合判断

**要点**：
- 多个参数同时受控/不受控的情况
- 参数之间的相关性（如效价下降 + 纯度下降可能同因）
- 过程整体健康度评估

## 输出规范

### 必须包含的章节

1. **参数选择**：分析对象和数据量
2. **控制图**：CL/UCL/LCL + 规格限
3. **规则检测**：触发的规则和位置
4. **趋势/漂移**：回归分析结果
5. **综合评估**：过程健康度 + 建议

- ✅ 趋势图（含回归线 + 预测区间）

## 行为约束

### 必须做的

- ✅ 控制限必须从数据计算，不能用规格限代替
- ✅ 规则检测必须标注触发的具体规则编号和批次
- ✅ 趋势分析必须报告回归斜率和 R²
- ✅ 数据量 < 20 点时结论加限定语

### 禁止做的

- ❌ 不要把规格限当作控制限
- ❌ 不要在不受控状态下计算 Cpk 并作为有效结论
- ❌ 不要只看单条规则而忽略其他规则的触发
- ❌ 不要忽略 WE-7（数据分层）——它可能说明测量系统有问题

## 常见错误防范

| 错误场景 | 错误做法 | 正确做法 |
|---------|---------|---------|
| 用规格限 32-50 U 画控制图 | "UCL=50, LCL=32" | 从数据计算控制限（可能更窄） |
| 触发 WE-3 趋势 | "有趋势" | 报告斜率、R²、预测何时超标 |
| 15 批数据 | "过程受控" | "数据量有限（15 批），仅供参考" |
| WE-7 触发 | 忽略 | "数据分层，可能测量系统分辨率不足" |
| 两个参数同时异常 | 分别分析 | 检查相关性，可能共同根因 |
