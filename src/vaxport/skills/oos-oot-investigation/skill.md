## 领域知识（Agent 必须知道）

### FDA OOS 指南核心要求（2006）

OOS 调查**必须**分两阶段进行，顺序不可颠倒：

```
OOS 结果产生
    │
    ▼
Phase I: 实验室调查（排除实验室误差）
    │
    ├── 发现实验室误差 → 无效结果，复检通过 → 关闭
    │
    └── 未发现实验室误差 → 进入 Phase II
                              │
                              ▼
                    Phase II: 全面调查（含生产调查）
                              │
                              ├── 确认生产OOS → 评估影响范围 → 处置决策
                              │
                              └──  inconclusive → 扩展调查
```

### Phase I: 实验室调查要点

检查项目（按优先级）：
1. **计算错误**：重新核算原始数据
2. **标准品问题**：标准品配制、稀释是否正确
3. **仪器设备**：校准状态、功能是否正常
4. **试剂/培养基**：是否在有效期内、批次是否正常
5. **操作过程**：是否按 SOP 执行、取样是否规范
6. **环境因素**：温湿度、洁净度是否符合要求

**关键约束**：
- Phase I 未确认实验室误差前，**不得复检**
- 如果 Phase I 确认了实验室误差，原结果无效，用新样品复检
- 复检次数应有上限（通常≤3次），防止"测到合格为止"

### Phase II: 全面调查要点

- 扩展到生产记录审查（批记录、工艺参数、设备状态）
- 评估对**其他批次**的影响
- 确定 root cause → CAPA → 处置决策

### 处置决策矩阵

| 调查结果 | 处置 | 说明 |
|---------|------|------|
| 确认实验室误差 | 原结果无效，复检放行 | 需有明确证据 |
| 确认生产OOS | 拒收或返工 | 产品质量受影响 |
| 无法确定原因 | 有条件放行 | 需加强监控+额外检测 |
| 假阳性 | 原结果无效，放行 | 需有充分证据排除 |

### OOT 规则体系

#### Western Electric 规则（8 条）

| 规则 | 描述 | 严重程度 |
|------|------|---------|
| Rule 1 | 1 点超出 3σ | Action |
| Rule 2 | 连续 9 点在中心线同侧 | Alert |
| Rule 3 | 连续 6 点递增或递减 | Alert |
| Rule 4 | 连续 14 点交替上下 | Alert |
| Rule 5 | 3 点中 2 点超出 2σ（同侧） | Action |
| Rule 6 | 5 点中 4 点超出 1σ（同侧） | Alert |
| Rule 7 | 连续 15 点在 1σ 内（分层问题） | Alert |
| Rule 8 | 连续 8 点在 1σ 外（两侧） | Alert |

#### Nelson 补充规则

| 规则 | 描述 |
|------|------|
| Nelson 5 | 2/3 点超过 2σ（同侧） |
| Nelson 6 | 4/5 点超过 1σ（同侧） |
| Nelson 7 | 连续 15 点在 ±1σ（数据分层） |
| Nelson 8 | 连续 8 点在 ±1σ 外 |

### 趋势 vs 漂移

- **趋势（Trend）**：参数随时间单调递增或递减（回归斜率显著）
- **漂移（Shift）**：参数均值突然变化到一个新水平（前后两段均值差异显著）

两者处理方法不同：趋势需找渐变原因，漂移需找突变事件。

## 分析框架（按此顺序执行）

### 第一步：OOS 总览

**目标**：获取 OOS 全貌

**要点**：
- OOS 总数、实验室/生产分类统计
- 按产品、检测类型、时间维度分布
- disposition 分布（released/conditional/rejected）
- 实验室 OOS 占比（正常应 > 40%，过低说明 Phase I 调查不充分）

**数据获取方向**：查询 oos_records 表

### 第二步：Phase I/II 调查分析

**目标**：评估调查质量

**要点**：
- Phase I 确认实验室误差的案例（假阳性）
- Phase II 确认生产 OOS 的案例
- 根因分类统计（取样问题、设备问题、工艺问题、试剂问题）
- 复检通过率和复检失败率

**分析方向**：评估调查的完整性和合规性

### 第三步：OOT 趋势分析

**目标**：识别趋势异常

**要点**：
- 已记录的 OOT 事件总览
- 按规则类型统计（WE Rule 1-8, Nelson）
- alert vs action 分布
- 误报率分析（评估为正常波动的比例）

**数据获取方向**：查询 oot_records 表

### 第四步：关联分析

**目标**：深度挖掘

**要点**：
- OOS 与 deviation 的关联（哪些 OOS 触发了偏差）
- 同一批次是否同时有 OOS + OOT
- 产品/设备/时间维度的聚集性分析
- 实验室 OOS 是否与特定分析师相关

### 第五步：结论与建议

**目标**：合规性评估 + 改进建议

**要点**：
- OOS 调查流程合规性评估
- 高风险 OOS 类型识别
- OOT 预警有效性评估
- 引用 FDA OOS 指南条款

## 输出规范

### 必须包含的章节

1. **OOS 总览**：分类统计、disposition 分布
2. **调查分析**：Phase I/II 调查结果、根因分类
3. **OOT 分析**：规则触发统计、误报率
4. **关联分析**：OOS-deviation 关联、聚集性
5. **结论与建议**：合规评估 + 改进建议

## 行为约束

### 必须做的

- ✅ OOS 调查必须区分 Phase I（实验室）和 Phase II（生产）
- ✅ 实验室 OOS 不能直接当作生产 OOS 处理
- ✅ OOT alert 需要评估是否为误报
- ✅ 复检次数必须有记录，超过 3 次需要特别说明

### 禁止做的

- ❌ 不要在 Phase I 未完成时直接复检
- ❌ 不要把"测到合格"当作有效调查
- ❌ 不要忽略实验室 OOS 占比过低的异常情况
- ❌ 不要把不同产品的 OOS 混在一起分析根因

## 常见错误防范

| 错误场景 | 错误做法 | 正确做法 |
|---------|---------|---------|
| OOS 结果出来直接复检 | 跳过 Phase I | 先完成实验室调查再决定 |
| 实验室 OOS 占比 0% | "所有 OOS 都是生产问题" | 质疑 Phase I 调查充分性 |
| OOT alert 全部确认为异常 | 无差别处理 | 逐条评估，区分真异常和误报 |
| 复检 5 次才合格 | "最终合格" | 质疑"测到合格"的合规性 |
