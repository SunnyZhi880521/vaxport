## 领域知识（Agent 必须知道）

### ICH Q9 质量风险管理框架

```
风险评估 → 风险控制 → 风险沟通 → 风险回顾
    │
    ├── 风险识别：可能发生什么？
    ├── 风险分析：发生概率和影响？
    ├── 风险评价：风险是否可接受？
    │
    └── 风险控制：如何降低风险？
        ├── 风险降低：采取措施降低 S/P/D
        └── 风险接受：残余风险可接受则放行
```

### FMEA 评分标准

#### 严重度 (Severity, S) — 1-10

| 分数 | 含义 |
|------|------|
| 1-2 | 无影响或几乎无影响 |
| 3-4 | 轻微影响，不影响产品质量 |
| 5-6 | 中等影响，可能影响中间产品质量 |
| 7-8 | 严重影响，影响最终产品质量 |
| 9-10 | 危及患者安全或产品报废 |

#### 发生概率 (Probability, P) — 1-10

| 分数 | 含义 |
|------|------|
| 1-2 | 极少发生（< 1/年） |
| 3-4 | 偶尔发生（1-5 次/年） |
| 5-6 | 有时发生（5-20 次/年） |
| 7-8 | 经常发生（> 20 次/年） |
| 9-10 | 几乎必然发生 |

#### 可检测性 (Detectability, D) — 1-10

| 分数 | 含义 |
|------|------|
| 1-2 | 几乎确定能检测到 |
| 3-4 | 大概率能检测到 |
| 5-6 | 有一定概率检测不到 |
| 7-8 | 大概率检测不到 |
| 9-10 | 几乎不可能检测到 |

### RPN 计算与判定

```
RPN = S × P × D

范围：1 ~ 1000
```

| RPN 范围 | 风险等级 | 行动 |
|---------|---------|------|
| < 50 | 低 | 常规监控 |
| 50-100 | 中 | 制定缓解措施 |
| > 100 | 高 | 必须采取缓解措施，降低至可接受水平 |

**关键约束**：
- RPN > 100 **必须**有缓解措施
- 残余 RPN（residual_rpn）> 50 说明缓解不充分
- 严重度 S ≥ 9 的项即使 RPN 不高也需要特别关注

### 变更分类

| 类型 | 说明 | 典型验证要求 |
|------|------|------------|
| equipment | 设备变更 | IQ/OQ/PQ |
| process | 工艺参数变更 | 工艺验证 |
| material | 物料变更 | 物料验证 |
| packaging | 包装变更 | 包装验证 |

### 变更生命周期

```
发起 → 风险评估 → 审批 → 实施 → 验证 → 关闭
                        │
                        └── 被拒绝（记录原因）
```

## 分析框架（按此顺序执行）

### 第一步：变更总览

**目标**：获取变更控制全貌

**要点**：
- 变更总数、按类型/风险等级/审批状态分布
- 完成率（closed + approved / total）
- 拒绝率及拒绝原因
- 超期完成率（actual_completion > target_completion）

**数据获取方向**：查询 change_control_records 表

### 第二步：风险评估分析

**目标**：评估 FMEA 执行质量

**要点**：
- RPN 分布（低/中/高）
- 高风险变更（RPN > 100）的缓解措施
- 残余 RPN 分析：缓解后是否降至可接受水平
- S ≥ 9 的项是否全部有缓解措施

**数据获取方向**：查询 change_control_risk_assessment 表

### 第三步：变更与 CAPA 关联

**目标**：评估变更驱动因素

**要点**：
- 由 CAPA 驱动的变更比例
- 变更是否有效解决了原始问题
- 变更引发的新偏差（变更有效性）

**分析方向**：关联 change_control_records 与 capa_records

### 第四步：设备变更影响

**目标**：评估设备变更对生产的影响

**要点**：
- 设备变更后的验证完成情况
- 变更前后关键工艺参数对比
- 设备变更后的产品合格率变化

**分析方向**：关联 change_control_records 与 equipment_master

### 第五步：结论与建议

**目标**：变更管理体系评估

**要点**：
- 变更控制流程合规性
- 高风险变更的闭环管理
- 残余风险是否可接受
- 引用 ICH Q9 条款

## 输出规范

### 必须包含的章节

1. **变更总览**：分类统计、完成率
2. **风险评估**：RPN 分布、高风险项处理

## 行为约束

### 必须做的

- ✅ RPN > 100 必须检查是否有缓解措施
- ✅ 残余 RPN > 50 必须标记为"缓解不充分"
- ✅ S ≥ 9 的项需要单独评估，不受 RPN 值限制
- ✅ 超期变更需要分析原因

### 禁止做的

- ❌ 不要只看 RPN 总值而忽略单项高分（如 S=10 即使 P×D 低也需关注）
- ❌ 不要把"已批准"等同于"风险已消除"
- ❌ 不要忽略被拒绝的变更（需要分析拒绝原因是否合理）
- ❌ 不要在没有验证数据的情况下确认变更有效

## 常见错误防范

| 错误场景 | 错误做法 | 正确做法 |
|---------|---------|---------|
| RPN = 120 无缓解 | "风险可接受" | "RPN > 100，必须有缓解措施" |
| 初始 RPN 200 → 残余 80 | "缓解有效" | "残余仍 > 50，缓解不充分" |
| S = 9, P = 2, D = 3 (RPN=54) | "中风险，常规处理" | "S=9 高危，需特别关注" |
| 变更已批准但无验证 | "变更已完成" | "缺少验证环节，变更未闭环" |
